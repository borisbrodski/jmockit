Getting started with the JMockit Testing Toolkit

Follow these instructions to start using JMockit:

  1. Download the full distribution for the latest release.
  2. Unpack the zip file to any directory. This will create the jmockit directory containing the library jars and source, samples, Javadocs and additional documentation.
  3. Add the library file jmockit.jar to your classpath (in your Java IDE project, Ant build file, and so on, as required). If you are using a JDK of version 1.6 or newer, also add <jdkDir>/lib/tools.jar to the classpath (<jdkDir> is the home directory for your local JDK installation). This second jar file isn't required, but it avoids the need to specify the initialization parameter -javaagent:jmockit.jar to the JVM when running tests.
  4. Optionally, configure your IDE to find API source and documentation in this same jmockit.jar file. Javadoc documentation in HTML is also available at jmockit/www/javadoc.

From here, you should probably proceed to the Tutorial. Check also the hundreds of JUnit sample tests available under the jmockit/samples folder, most of which use the JMockit APIs.
