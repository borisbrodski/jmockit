Getting started with the JMockit Testing Toolkit

Follow these instructions to start using JMockit:

  1. Download the full distribution for the latest release.
  2. Unpack the zip file to any directory. This will create the jmockit directory containing the library jars and source, samples, Javadocs and additional documentation.
  3. Add the library file jmockit.jar to your classpath (in your Java IDE project, Ant build file, and so on, as required). If you are using a JDK of version 1.6 or newer, also add <jdkDir>/lib/tools.jar to the classpath (<jdkDir> is the home directory for your local JDK installation). This second jar file isn't required, but it avoids the need to specify the initialization parameter -javaagent:jmockit.jar to the JVM when running tests.
  4. Optionally, configure your IDE to find API source and documentation in this same jmockit.jar file. Javadoc documentation in HTML is also available at jmockit/www/javadoc.

If your project uses Maven, then add the following dependency to your pom.xml file: 

      <dependency>
         <groupId>mockit</groupId>
         <artifactId>jmockit</artifactId>
         <version>0.992</version>
         <scope>test</scope>
      </dependency>

The JMockit artifacts, including a binary jar and a sources jar for each release, are located in the Java.net Maven 2 Repository.

You might also need to add the following to your Maven configuration: 

      <repository>
         <id>maven2-repository.dev.java.net</id>
         <name>Java.net Repository for Maven</name>
         <url>http://download.java.net/maven/2/</url>
      </repository>

If using JDK 1.6, you will probably also want to have the following dependency in your project: 

      <dependency>
         <groupId>com.sun</groupId>
         <artifactId>tools</artifactId>
         <version>1.6.0</version>
         <scope>system</scope>
         <systemPath>${java.home}/../lib/tools.jar</systemPath>
      </dependency>

From here, you should probably proceed to the Tutorial. Check also the hundreds of JUnit sample tests available under the jmockit/samples folder.
