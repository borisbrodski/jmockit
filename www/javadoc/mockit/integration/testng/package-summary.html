Package mockit.integration.testng Description

Provides integration with TestNG test runners, for versions 5.14+. Contains the Initializer test listener class that can be specified to TestNG at startup, as well as the "startup mock" implementation for integration with the TestNG test runner.

This integration provides the same benefits to test code as the one for JUnit 4:

  1. Unexpected invocations specified through the Annotations or the Expectations API are automatically verified just before the execution of a test ends (specifically, immediately after the execution of the test method).
  2. Any mock classes applied with the Annotations API from inside a method annotated as a @Test or a @BeforeMethod will be discarded right after the execution of the test or the test method, respectively.
  3. Any mock class set up for the whole test class (either through a call to Mockit.setUpMocks(java.lang.Object...) from inside a @BeforeClass method, or by annotating the test class with UsingMocksAndStubs) will only apply to the tests in this same test class. That is, you should not explicitly tell JMockit to restore the mocked classes in an @AfterClass method.
  4. Test methods will accept mock parameters, whose values are mocked instances automatically created by JMockit and passed by the test runner when the test method gets executed.
