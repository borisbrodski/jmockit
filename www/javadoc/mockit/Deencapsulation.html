Class Deencapsulation

java.lang.Object
  mockit.Deencapsulation

public final class Deencapsulation
extends java.lang.Object

Provides utility methods that enable access to (ie "de-encapsulate") otherwise non-accessible fields, methods and constructors belonging to code under test.

The Expectations class (and its subclasses) provide context-specific versions of this same methods. So, when dealing with mocked types through the Expectations API there is usually no need to import this class. If used it will work just as well, though.

Tutorial
Sample tests: DeencapsulationTest, ServiceHolder_JMockit_Test, ScreenTransitionContainerResizeTest
